## 维护规则
- **新发现一条横切契约**（周期性体检 / 评审中撞见）→ 加一行，状态先标「无 gate=挂账」，再排是否 gate 化。
- **一条契约建成 gate** → 更新「机器强制点」列写清 gate 位置与运行命令、状态改 ✅。
- **一条契约评估后确认机械不可行** → 状态保持「无 gate=挂账」，并在对应 openspec/注释里锁定「靠人 + 台账提醒」的决策，不硬造脆弱测试。

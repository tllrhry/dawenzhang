# 服务器更新手册

本文适用于当前演示服务器的宿主机部署方式，不适用于本地 `./start.sh` 或 Docker Compose 开发流程。

## 当前部署结构

| 组件 | 当前方式 |
| --- | --- |
| 源码仓库 | `/root/dawenzhang`，跟踪 `origin/main` |
| 后端 | systemd 单元 `dawenzhang.service`，`User=dawenzhang`，`WorkingDirectory=/root/dawenzhang/backend`，`EnvironmentFile=/root/dawenzhang/.env`，`PYTHONPATH=/root/dawenzhang/backend`，Uvicorn 监听 `127.0.0.1:8000` |
| 前端 | 在源码目录构建，发布至 `/opt/dawenzhang/frontend` |
| Web 入口 | 宿主机 Nginx，负责静态文件与 `/api/` 反向代理 |
| 数据库 | 宿主机 PostgreSQL + pgvector，**Unix socket + peer 认证**：DB 命令必须以 `dawenzhang` 用户运行，`root` 直连会报 `role "root" does not exist` |
| 运行配置 | `/root/dawenzhang/.env`，不得提交、复制或打印 |
| 运行时资产 | 模板与映射 Excel 放 **`/opt/dawenzhang/assets/`**，由 `.env` 中的绝对路径变量指向；**不随 git 分发**（仓库 `.gitignore` 忽略 `模板文件/`、`五篇大文章映射/`、`国民经济/`） |

服务器地址、登录凭据和访问入口由运维渠道保管，不写入本仓库。登录改用 SSH 密钥免密（见「凭据与免密登录」一节），不要把密码写进任何仓库文件。

### 运行时资产与路径变量

`git pull` **只带代码，不带运行资产**。所有模板/映射/目录 Excel 都在 `.gitignore` 内，必须单独 `scp` 到 `/opt/dawenzhang/assets/`，并在 `.env` 里用**绝对路径**指向；否则应用回落到相对默认路径（相对 `WorkingDirectory=/root/dawenzhang/backend`）而找不到文件。

| `.env` 变量 | 指向 | 谁读取 |
| --- | --- | --- |
| `NATIONAL_ECONOMY_TEMPLATE_PATH` | `/opt/dawenzhang/assets/国民经济类别模版.docx` | 后端服务（模板下载） |
| `NATIONAL_ECONOMY_CATALOG_PATH` | `/opt/dawenzhang/assets/国民经济行业分类V2.xlsx` | 目录同步命令；目录入库后运行时不读 Excel |
| `TECHNOLOGY_FINANCE_TEMPLATE_PATH` | `/opt/dawenzhang/assets/科技金融模版.docx` | 后端服务（模板下载） |
| `GREEN_FINANCE_TEMPLATE_PATH` / `DIGITAL_FINANCE_TEMPLATE_PATH` / `PENSION_FINANCE_TEMPLATE_PATH` | 对应场景模板 | 后端服务（模板下载） |
| `AGRICULTURE_RELATED_TEMPLATE_PATH` | `/opt/dawenzhang/assets/涉农类别模版.docx` | 后端服务（涉农模板下载与上传校验） |
| `FIVE_ARTICLES_MAPPING_SOURCE_PATH` | `/opt/dawenzhang/assets/贷款投向-五篇大文章映射表.xlsx` | 四个可执行五篇场景的唯一映射同步源（Stage B 运行时读 DB，不读文件） |

目录同步必须使用 V2 十列表头资产；同步完成后，五篇映射同步会以最新 V2 目录做 2/3/4 位 code/name 校验。旧 `TECHNOLOGY_FINANCE_MAPPING_PATH`、`GREEN_FINANCE_MAPPING_PATH`、`DIGITAL_FINANCE_MAPPING_PATH`、`PENSION_FINANCE_MAPPING_PATH` 及四份场景专属映射 Excel 均不再被运行时读取。

## 标准更新步骤

以有权限的管理员账户登录服务器后执行。更新前请确保当前发布窗口允许短暂重启后端。

```bash
cd /root/dawenzhang

# 1. 先确认服务器没有人工未提交修改，再拉取 main。
git status --short --branch
export GIT_PAGER=cat
timeout 60 git -c protocol.version=2 -c http.version=HTTP/1.1 fetch origin main
git pull --ff-only origin main

# 2. 使用与 systemd 相同的配置和服务用户执行迁移。
#    不要以 root 直接运行 migrate.sh：本机 PostgreSQL 认证依赖 dawenzhang 用户。
set -a
. /root/dawenzhang/.env
set +a
export PATH=/opt/dawenzhang/venv/bin:$PATH
runuser -u dawenzhang --preserve-environment -- bash backend/scripts/migrate.sh

# 3. 安装锁定的前端依赖、构建并发布静态文件。
npm --prefix frontend ci
npm --prefix frontend run build
rsync -a --delete --chown=dawenzhang:dawenzhang \
  frontend/dist/ /opt/dawenzhang/frontend/

# 4. 校验 Nginx 配置，重启后端并重载 Nginx。
nginx -t
systemctl restart dawenzhang.service
systemctl reload nginx.service
```

`git pull --ff-only` 若提示不能快进，或 `git status` 显示服务器本地改动，请停止更新并先处理差异；不要用 `reset --hard` 覆盖服务器文件。

## 发布后验证

```bash
# 代码与远端 main 一致，且工作区无修改。
cd /root/dawenzhang
git status --short --branch
git rev-parse HEAD
git rev-parse origin/main

# 两项服务均应输出 active。
systemctl is-active dawenzhang.service
systemctl is-active nginx.service

# API 健康检查应返回 status=ok，且 database.pgvector=enabled。
curl --fail --silent http://127.0.0.1:8000/api/v1/health
```

最后使用正常浏览器访问入口完成一次前端检查。当前站点有访问拦截，未获授权直接请求页面可能返回 404；这不代表前端发布失败。

## 国民经济 V2 目录与五篇映射发布

五篇场景的 Stage B 映射依赖 V2 目录。因此发布顺序固定为：**迁移 → 同步 V2 目录 → 用同一份合并映射表按场景分别发布科技、绿色、数字、养老**。普惠金融不使用本流程。

服务器发布（按序执行）：

```bash
# A. 本机 → 服务器：上传 V2 目录、唯一合并映射表和四份模板（资产不随 git 分发）
scp "国民经济/国民经济行业分类V2.xlsx" root@<server>:/opt/dawenzhang/assets/国民经济行业分类V2.xlsx
scp "五篇大文章映射/贷款投向-五篇大文章映射表.xlsx" root@<server>:/opt/dawenzhang/assets/贷款投向-五篇大文章映射表.xlsx
scp "模板文件/五篇大文章/"*.docx root@<server>:/opt/dawenzhang/assets/
ssh root@<server> 'chown dawenzhang:dawenzhang /opt/dawenzhang/assets/*'

# 以下在服务器上执行
cd /root/dawenzhang && export GIT_PAGER=cat

# B. 更新为唯一目录/映射路径（绝对路径；保留模板路径，不保留四个旧映射路径）
sed -i '/^NATIONAL_ECONOMY_CATALOG_PATH=/d;/^FIVE_ARTICLES_MAPPING_SOURCE_PATH=/d' .env
printf '%s\n' 'NATIONAL_ECONOMY_CATALOG_PATH=/opt/dawenzhang/assets/国民经济行业分类V2.xlsx' \
  'FIVE_ARTICLES_MAPPING_SOURCE_PATH=/opt/dawenzhang/assets/贷款投向-五篇大文章映射表.xlsx' >> .env

# C. 拉代码
git pull --ff-only origin main

# D. 迁移（以 dawenzhang 用户跑）
set -a; . /root/dawenzhang/.env; set +a
export PATH=/opt/dawenzhang/venv/bin:$PATH
runuser -u dawenzhang --preserve-environment -- bash backend/scripts/migrate.sh

# E. 【关键】先全量同步 V2 目录；新 V2 source hash 必须创建新 catalog version 并重建 embedding。
runuser -u dawenzhang -- bash -c 'cd /root/dawenzhang && set -a && . ./.env && set +a && \
  PYTHONPATH=backend /opt/dawenzhang/venv/bin/python backend/scripts/sync_national_economy_catalog.py'

# F. 用同一合并表按场景分别预检/原子发布；任一场景失败不影响其他场景已发布版本。
for scenario_id in technology_finance green_finance digital_finance pension_finance; do
  runuser -u dawenzhang -- bash -c "cd /root/dawenzhang && set -a && . ./.env && set +a && \
    PYTHONPATH=backend /opt/dawenzhang/venv/bin/python backend/scripts/sync_five_articles_mapping.py $scenario_id" || exit 1
done

# G. 前端构建发布 + 重启（同标准步骤）
npm --prefix frontend ci && npm --prefix frontend run build
rsync -a --delete --chown=dawenzhang:dawenzhang frontend/dist/ /opt/dawenzhang/frontend/
nginx -t && systemctl restart dawenzhang.service && systemctl reload nginx.service
```

上线验证：

```bash
# 四个五篇场景应为 available
curl --fail --silent http://127.0.0.1:8000/api/v1/scenarios
# 映射均已发布；按场景核对行数与 4/3/2 位抽样 code/name。
runuser -u dawenzhang -- bash -c 'cd /root/dawenzhang && set -a && . ./.env && set +a && \
  PYTHONPATH=backend /opt/dawenzhang/venv/bin/python -c "
from app.db.session import get_sessionmaker
from sqlalchemy import text
with get_sessionmaker()() as s:
    print(s.execute(text(\"select scenario_id, count(*) from five_articles_mapping_rows group by scenario_id order by scenario_id\")).all())"'
```

最后用真实企业 Word 在浏览器走一遍上传→分类，确认出「一致 / 不一致」结论（会真实调用 DeepSeek Stage B）。

### 向量库整体替换（目录编码规则发生变化时）

当国民经济目录从旧的“大类编码 + 小类编码”升级为“大类 + 中类 + 小类”时，不能只向云端追加或部分替换向量片段。旧版本向量与新版本目录会同时参与检索，导致召回结果混用。此时必须将本地已经完成向量化的目录库整体打包并替换云端对应两张表：

```bash
# 本地数据库容器中导出完整目录版本表和向量片段表。
docker exec dawenzhang-db-1 pg_dump -U dawenzhang -d dawenzhang \
  --format=custom --compress=6 --data-only \
  --table=public.national_economy_catalog_versions \
  --table=public.national_economy_industry_chunks \
  -f /tmp/dawenzhang-vector-library.dump
docker cp dawenzhang-db-1:/tmp/dawenzhang-vector-library.dump /tmp/
scp /tmp/dawenzhang-vector-library.dump <server>:/tmp/

# 云端先停后端；可先保留一份临时备份。
systemctl stop dawenzhang.service
runuser -u dawenzhang -- pg_dump -d dawenzhang --format=custom --compress=6 --data-only \
  --table=public.national_economy_catalog_versions \
  --table=public.national_economy_industry_chunks \
  -f /tmp/cloud-vector-library-before-replace.dump

# 先清空两张向量库表，再按 dump 的父表/子表顺序恢复。
runuser -u dawenzhang -- psql -d dawenzhang -v ON_ERROR_STOP=1 -c \
  "TRUNCATE TABLE national_economy_industry_chunks, national_economy_catalog_versions RESTART IDENTITY CASCADE"
runuser -u dawenzhang -- pg_restore -d dawenzhang --data-only \
  /tmp/dawenzhang-vector-library.dump

# 恢复后校正自增序列，再启动服务并检查健康状态。
runuser -u dawenzhang -- psql -d dawenzhang -v ON_ERROR_STOP=1 -c \
  "SELECT setval(pg_get_serial_sequence('national_economy_catalog_versions','id'), COALESCE(MAX(id), 1), MAX(id) IS NOT NULL) FROM national_economy_catalog_versions;
   SELECT setval(pg_get_serial_sequence('national_economy_industry_chunks','id'), COALESCE(MAX(id), 1), MAX(id) IS NOT NULL) FROM national_economy_industry_chunks;"
systemctl start dawenzhang.service
```

恢复后必须核对本地与云端的目录版本、`source_hash`、每个 `catalog_version_id` 的片段数，并确认五篇映射表行数未变化。确认服务正常后再删除 `/tmp` 下的传输包和备份。

### 踩坑记录（2026-07-14）

- 服务器执行 `git fetch origin main` 即使加了 protocol v2 和 HTTP/1.1 仍可能因 GitHub TLS/peer 网络错误失败。此时不要使用 `reset --hard` 或手动覆盖服务器源码；在本地执行 `git bundle create /tmp/dawenzhang-main.bundle main`，上传到服务器后执行 `git fetch /tmp/dawenzhang-main.bundle main:refs/remotes/origin/main`，再在服务器执行 `git merge --ff-only origin/main`。
- 服务器普通应用用户执行 `pg_restore --disable-triggers` 会因无权操作 PostgreSQL 系统触发器报错。向量表已先清空时，dump 会按目录版本父表、向量片段子表顺序恢复，因此不需要 `--disable-triggers`；恢复后仍要检查两张表的实际行数。
- 国民经济向量库编码规则变更时，部分删除、部分追加会留下旧版本向量并造成混合召回。本流程必须整体清空并导入本地已向量化数据，不能改成增量替换。

>
> 绿色、数字、养老金融上线实测记录（2026-07-14）：迁移已到 `0009_five_articles_integrity`；三套模板、映射文件和绝对路径变量均已配置，映射分别发布为 `green_finance` 188 行、`digital_finance` 137 行、`pension_finance` 229 行（均为 version 1）。三条真实 DeepSeek 闭环分别命中 `3411`（源行 2）、`3914`（源行 5）、`8514`（源行 17），均为 completed/consistent，历史与场景专属导出工作表均已核验。

## 凭据与免密登录

- **绝不把服务器地址、账号、密码写入仓库任何文件**（含本手册、脚本、`.env` 示例、提交信息）。`docs/` 会被 git 跟踪并 push 到 GitHub，一旦写入即等于公开泄露。
- 登录使用 **SSH 密钥免密**：本机生成密钥对，公钥装到服务器 `~/.ssh/authorized_keys`，在本机 `~/.ssh/config` 配一个 Host 别名（如 `dawenzhang-prod`）。之后 `ssh dawenzhang-prod` / `scp ... dawenzhang-prod:...` 全程无需密码，也无需把密码贴进终端。
- `~/.ssh/config` 与私钥都在本机、不进仓库；服务器密码在完成密钥登录后应更换并离线保管。

## 常见问题

### `python: not found`

服务器的项目 Python 位于 `/opt/dawenzhang/venv/bin`。在执行迁移前运行：

```bash
export PATH=/opt/dawenzhang/venv/bin:$PATH
```

### 迁移报 `role "root" does not exist`

迁移以 root 用户连接了本机 PostgreSQL。按标准步骤加载 `.env` 后，以 `dawenzhang` 用户运行：

```bash
runuser -u dawenzhang --preserve-environment -- bash backend/scripts/migrate.sh
```

### 前端构建报 `tsc: not found`

前端依赖尚未安装或已被清理，执行：

```bash
npm --prefix frontend ci
```

之后重新运行构建、`rsync`、Nginx 校验和服务重启步骤。

### `git fetch` 卡住或报 `GnuTLS recv error (-110)`

服务器到 GitHub 的普通 HTTPS 请求可能正常，但 Git 的上传包请求会偶发卡住或在 TLS 连接被异常中断时失败。不要让命令无限等待；使用标准步骤中的带超时命令重试：

```bash
export GIT_PAGER=cat
timeout 60 git -c protocol.version=2 -c http.version=HTTP/1.1 fetch origin main
git pull --ff-only origin main
```

若超时或失败，等待片刻后重试。`git status --short --branch` 确认仍处于旧版本且工作区干净后再试；不要因网络错误使用 `reset --hard` 或手动覆盖源码。

### Git 分页器阻断了后续命令

SSH 交互终端中执行 `git log` 等命令时，分页器可能显示“Press RETURN to continue”，随后粘贴的命令会被当作分页器输入。发布流程开始前设置：

```bash
export GIT_PAGER=cat
```

需要查看历史时也使用 `git --no-pager log --oneline`。若已进入分页器，先退出或按回车回到 shell，再逐条确认并重新执行未运行的命令。

### 后端重启失败或健康检查不通过

不要继续重载其他配置。先收集最近日志并检查单元状态：

```bash
systemctl --no-pager --full status dawenzhang.service
journalctl -u dawenzhang.service -n 100 --no-pager
```

确认 `.env`、PostgreSQL 和迁移状态后再处理；不要将日志中的密钥内容复制到 issue、聊天或仓库。
